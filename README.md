# Customer Churn Prediction — Telecom
## Problem Statement
Customer churn is one of the most critical challenges in the telecom industry. Retaining an existing customer is significantly cheaper than acquiring a new one. This project builds an ML-based solution to predict which customers are likely to churn, enabling proactive retention strategies.
## Objective
To predict customer churn probability using historical behavioral and account data, and translate model insights into actionable business recommendations.
## Approach

Performed extensive Exploratory Data Analysis (EDA) to identify patterns and churn signals
Applied feature engineering to extract meaningful variables driving churn behavior
Built and compared multiple ML models, selecting the best performer based on key metrics
Deployed the final model as a Flask-based web application for real-time churn predictions

## Key Business Insights

Identified top drivers of churn including contract type, tenure, and service usage patterns
Segmented high-risk customers to enable targeted retention interventions
Linked data findings to business impact — reducing churn directly improves revenue and LTV

## Product Thinking
If deployed at scale, this model can power:

Early warning systems for customer success teams
Personalized retention offers triggered by churn probability scores
Dashboards tracking churn risk across customer segments in real time

### Tech Stack
Python, Pandas, Scikit-learn, Flask, Matplotlib, Seaborn
